HARPA AI - Zero-Click Account Takeover

Vulnerability: Web Cache Deception (WCD)

Attack Vector: Cloudflare CDN caches authenticated HTML responses when a static file extension is appended to library page URLs.

Impact: Any logged-in harpa.ai user who loads a poisoned URL (via <img>, <link>, or iframe on any website) has their JWT token cached on Cloudflare's CDN. The attacker then fetches the cached page to steal the JWT.

Zero-Click: No user interaction required. The poisoned URL can be embedded as a hidden resource on any webpage.

Step 1: Poison the victim's cache (simulated)

In a real attack, this would be an invisible <img> tag on any website the victim visits:

<img src="https://harpa.ai/library/deep-research-bot/6728a0a922b006c16f7f44e5.css" style="display:none">

Or a CSS preload:

<link rel="prefetch" href="https://harpa.ai/library/deep-research-bot/6728a0a922b006c16f7f44e5.css">
Waiting for victim cache poisoning...
